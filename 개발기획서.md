## 1. 프로젝트 개요

### 1.1 현황 분석

현재 사이트는 **순수 프론트엔드 랜딩페이지**로 구성되어 있으며, 아래와 같은 한계가 존재합니다.

| 구분 | 현황 | 문제점 |
|------|------|--------|
| 상담 신청 폼 | `setTimeout` 시뮬레이션 처리 | 실제 데이터 저장·알림 없음 |
| 관리자 기능 | 없음 | 문의 내용 확인 불가 |
| 분석/통계 | Vercel Analytics만 적용 | 전환율·유입 경로 추적 불가 |
| 콘텐츠 관리 | 하드코딩 | 코드 수정 없이 내용 변경 불가 |
| 예약 시스템 | 없음 | 코칭 일정 관리 불가 |

### 1.2 고도화 목표

1. **리드(Lead) 수집 자동화** — 상담 신청 폼 → DB 저장 + 관리자 카카오톡/이메일 알림
2. **관리자 대시보드** — 신청자 목록 조회·상태 관리·메모 기능
3. **콘텐츠 관리 시스템(CMS)** — 코드 없이 서비스·후기·FAQ 내용 수정
4. **예약 시스템** — 코칭 상담 일정 선택 및 확정
5. **성과 추적** — 폼 전환율 분석, UTM 파라미터 추적

## 2. 기술 아키텍처

### 2.1 전체 구조

```
[클라이언트 (Next.js)]
        ↕ API Routes (Next.js App Router)
[비즈니스 로직 레이어]
        ↕
[데이터베이스]     [외부 서비스]
 PostgreSQL         - 카카오 알림톡 / 이메일 (nodemailer)
 (Supabase 권장)    - 카카오 캘린더 API (예약)
                    - Google Analytics 4
```

### 2.2 기술 스택 선정

| 영역 | 채택 기술 | 선정 이유 |
|------|----------|----------|
| **BaaS / DB** | Supabase (PostgreSQL) | 무료 티어·실시간 기능·인증 내장 |
| **API** | Next.js Route Handlers (`/app/api/`) | 별도 서버 불필요·서버리스 배포 |
| **ORM** | Prisma | 타입 안전·마이그레이션 관리 용이 |
| **인증 (관리자)** | NextAuth.js v5 | Next.js 네이티브·세션 관리 |
| **이메일 알림** | Nodemailer + Gmail SMTP | 무료·빠른 구현 |
| **카카오 알림** | 카카오 비즈메시지 API | 높은 오픈율·한국 서비스 최적화 |
| **파일 스토리지** | Supabase Storage | DB와 동일 플랫폼·CDN 내장 |
| **유효성 검사** | Zod (이미 설치됨) | 프론트·백엔드 스키마 공유 |
| **배포** | Vercel | Next.js 최적 환경·CI/CD 자동화 |

## 3. 데이터베이스 설계

### 3.1 ERD 개요

```
consultations (상담 신청)
├── id (UUID, PK)
├── name (VARCHAR 50)
├── phone (VARCHAR 20)
├── interest (VARCHAR 100)
├── message (TEXT)
├── status (ENUM: pending/contacted/completed/cancelled)
├── admin_note (TEXT)
├── utm_source / utm_medium / utm_campaign
├── created_at (TIMESTAMPTZ)
└── updated_at (TIMESTAMPTZ)

bookings (예약 일정)
├── id (UUID, PK)
├── consultation_id (FK → consultations)
├── scheduled_at (TIMESTAMPTZ)
├── duration_minutes (INT, default 60)
├── meeting_type (ENUM: phone/zoom/in-person)
└── status (ENUM: pending/confirmed/cancelled)

admin_users (관리자 계정)
├── id (UUID, PK)
├── email (VARCHAR, UNIQUE)
├── name (VARCHAR)
└── created_at

contents (CMS: 동적 콘텐츠)
├── id (UUID, PK)
├── section (VARCHAR: services/testimonials/faq/about)
├── key (VARCHAR)
├── value (JSONB)
└── updated_at
```

## 4. API 설계

### 4.1 공개 API (인증 불필요)

| Method | Endpoint | 설명 |
|--------|----------|------|
| `POST` | `/api/consultations` | 상담 신청 접수 |
| `GET` | `/api/contents/{section}` | CMS 콘텐츠 조회 |
| `GET` | `/api/bookings/available-slots` | 예약 가능 슬롯 조회 |
| `POST` | `/api/bookings` | 예약 생성 |

### 4.2 관리자 API (인증 필요)

| Method | Endpoint | 설명 |
|--------|----------|------|
| `GET` | `/api/admin/consultations` | 신청 목록 조회 (페이지네이션·필터) |
| `PATCH` | `/api/admin/consultations/{id}` | 상태 변경·메모 저장 |
| `GET` | `/api/admin/stats` | 전환율·신청 현황 통계 |
| `PUT` | `/api/admin/contents/{id}` | CMS 콘텐츠 수정 |
| `GET` | `/api/admin/bookings` | 예약 목록 관리 |

### 4.3 상담 신청 API 상세 (`POST /api/consultations`)

**요청 Body (Zod 스키마 정의)**
```ts
const ConsultationSchema = z.object({
  name:     z.string().min(2).max(50),
  phone:    z.string().regex(/^010-\d{4}-\d{4}$/),
  interest: z.string().max(100).optional(),
  message:  z.string().max(1000).optional(),
  utm_source:   z.string().optional(),
  utm_medium:   z.string().optional(),
  utm_campaign: z.string().optional(),
});
```

**처리 흐름**
```
1. Zod 유효성 검사
2. 스팸 방지 (Honeypot 필드 + Rate Limiting)
3. DB 저장 (Supabase)
4. 관리자 알림 발송 (이메일 + 카카오 알림톡 병렬 처리)
5. 성공 응답 반환 (201 Created)
```
