## ✅ STEP 7 — 상담 신청 API Route (app/api/consultations/route.ts)

```
Next.js App Router의 Route Handler로 상담 신청 API를 구현해주세요.

파일 위치: app/api/consultations/route.ts

POST 메서드 처리:
1. Rate Limiting 체크 (IP당 분당 3회 제한, @upstash/ratelimit 사용)
   - Redis 없으면 "메모리 기반 간단한 제한"으로 대체 가능
2. Request body를 STEP 5의 Zod 스키마로 유효성 검사
3. Prisma로 consultation 레코드 DB 저장
4. sendConsultationAlert() 호출 (비동기, await 없이 fire-and-forget)
5. 201 응답 반환

에러 처리:
- 유효성 실패: 400 + 에러 상세
- Rate Limit 초과: 429
- 서버 에러: 500

TypeScript, NextResponse 사용
```

```
현재 contact.tsx의 handleSubmit 함수가 setTimeout 시뮬레이션으로 되어 있습니다.

현재 코드의 handleSubmit:
  const handleSubmit = async (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    setIsSubmitting(true);
    await new Promise((resolve) => setTimeout(resolve, 1000));
    setIsSubmitting(false);
    setIsSubmitted(true);
  };

수정 요구사항:
1. FormData에서 name, phone, interest, message 값 추출
2. fetch('/api/consultations', { method: 'POST', body: JSON.stringify({...}) }) 호출
3. URL 파라미터에서 utm_source, utm_medium, utm_campaign 자동 추출해서 함께 전송
4. 응답 성공(2xx) → setIsSubmitted(true)
5. 실패 → 에러 상태 추가하여 사용자에게 메시지 표시
6. 에러 상태 변수(isError, errorMessage) 추가 및 UI 반영
```
